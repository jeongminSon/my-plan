## 1. 현재 UI 감사 (located findings)

추측이 아니라 코드에서 확인한 문제만 적습니다.

### 렌즈 1 — 브랜드 정렬
- **브랜드 정체성이 없음.** `primary`가 일반적인 파랑(`#2f6fed`)이고, 로고/워드마크·브랜드 모먼트가 전무. "my-plan"이라는 이름값이 화면에 드러나지 않습니다.
- 다행히 브랜드(파랑)가 의미색(위험=빨강, 성공=초록, 우선순위=빨강/주황/초록)과 **겹치지 않음** → 색 충돌은 없음. 이 분리는 유지합니다.

### 렌즈 2 — 일관성 & 접근성
- **타입 스케일이 비정형.** 폰트 크기가 12·13·14·16·22·24·26·28로 컴포넌트마다 리터럴로 흩어져 있음(`headerTitle:28`, `statValue:24`, `title:26`, `count:14`, `memo:13`, `metaText:12`…). 명명된 스케일 토큰이 없습니다.
- **라운드/그림자 패밀리 없음.** radius가 8·10·12·14 혼재, elevation(그림자) 토큰은 아예 없어 카드가 평면적(배경색 차이로만 구분).
- **웹 포커스 링 부재.** `:focus-visible` 대응이 없어 키보드 사용자가 현재 포커스를 알 수 없음(react-native-web).
- **숫자에 tabular-nums 미적용.** "오늘 N개 중 M개", "{todayPct}%", 주간 완료 숫자가 자릿수에 따라 흔들립니다.
- **이모지를 아이콘으로 사용**(📅 🔁 ⏰ ☑ ⚑ 📁 👤 🌗). 플랫폼·OS마다 렌더가 다르고, 크기·색 통제가 불가하며, 스크린리더 읽힘이 들쭉날쭉합니다. → 가장 큰 일관성 결함.

### 렌즈 3 — 정보 밀도 & 스캔성
- **메타 칩(우선순위·목록·반복·알림·하위)이 전부 12px 회색 텍스트**로 나열돼, 설정됨/안됨 구분이 **색(파랑) 하나에만** 의존. 스캔이 어렵고 색각 보조 원칙 위반.
- 같은 메타 칩의 **터치 영역이 시각상 12px 텍스트**(hitSlop 6)로 44px 미달.
- 진행 통계 카드는 있으나 **"오늘"의 핵심 수치가 헤더 회색 14px**로 작게 묻혀 있음(글랜스 가치가 약함).

### 렌즈 4 — 즉시 인지(모바일 글랜스)
- 앱을 열었을 때 **가장 먼저·크게 보여야 할 "오늘 얼마나 했나"** 가 시각적으로 1순위가 아님. 헤더 타이틀("할일", 28px)이 더 큰데, 정작 사용자가 원하는 답은 진행률입니다.
- 체크박스가 유니코드 글리프(☐/☑, 22px)라 탭 타깃·정렬·브랜드감이 약함.

**긍정적으로 잘 된 것**(유지): 라이트/다크 토큰, AA 의식한 회색 단계, 빈 상태 문구, 마감 지남의 "지남" 라벨(색+텍스트 동시), 입력 16px(iOS 줌 방지), 인증 화면의 배너(아이콘+문구).

## 2. 디자인 컨셉 & 톤앤매너

### 컨셉: **"Calm Focus — 차분한 집중"**

할일 앱은 펼치는 순간 **압박이 아니라 정돈된 안도감**을 줘야 합니다. 화려함보다 **여백·명료한 위계·한눈에 들어오는 오늘**이 핵심입니다. 톤앤매너 세 줄 요약:

1. **조용한 배경, 또렷한 한 곳.** 큰 면적은 중립색으로 비우고, 브랜드 색은 "지금 누를 것"과 "오늘의 핵심 수치"에만 씁니다.
2. **숫자가 주인공.** "오늘 3 / 5 완료"가 화면에서 가장 크고 진한 요소. 나머지는 보조.
3. **부드럽지만 분명하게.** 둥근 라운드(12px), 아주 옅은 그림자, 굵은 라벨. 칩은 색이 아니라 **형태(채움 vs 외곽선)** 로 상태를 구분합니다.

### 브랜드 토큰 (인디고 블루 — 의미색과 비충돌)

기존 파랑을 정체성 있는 **인디고 블루**로 다듬고, 선택/강조용 틴트를 추가합니다. (정식 브랜드 가이드가 생기면 교체 — *confirm against official guideline*.)

```
라이트
--brand        #3B5BDB   주요 액션·활성 (흰 텍스트 대비 ≈5.0:1, AA 통과)
--brand-strong #2C46B8   hover/press·강조 텍스트
--brand-tint   #EEF1FE   선택 칩·브랜드 헤일로·진행바 트랙 위 채움 배경
--brand-border #D8DFFB   브랜드 외곽선/구분선

다크
--brand        #7C93FF
--brand-strong #9DB0FF
--brand-tint   #20283F
--brand-border #2E3A5C
--onPrimary(dark) #0B1020
```

의미색(위험/성공/우선순위)은 **그대로 유지** — 브랜드 인디고와 색이 겹치지 않습니다.

### 타입 스케일 (1.25 비율 + tabular 숫자)

```
fsXs 12 · fsSm 13 · fsMd 15 · fsLg 17 · fsXl 20 · fs2xl 24 · fs3xl 30 · fsHero 40
무게: 본문 500–600 · 라벨 700 · 숫자 800
숫자(카운트·%·주간완료)는 fontVariant:['tabular-nums'] 적용
폰트: Pretendard(한글 우선) + 시스템 폴백
```
**글랜스 규칙**: 오늘 화면에서 가장 중요한 값(오늘 완료 수/율)은 `fs3xl`(30) 이상 + 무게 800.

### 간격 · 라운드 · 그림자

```
간격(4px 베이스): 4 · 8 · 12 · 16 · 20 · 24 · 32
라운드: rSm 8 · rMd 12 · rLg 16 · rPill 999
그림자(아주 옅게, 카드·모달에만):
  shadowSm  카드 살짝 띄움
  shadowMd  팝오버/모달
```

### 아이콘 시스템
이모지를 걷어내고 **`@expo/vector-icons`(Feather/Ionicons)** 단일 세트로 통일합니다. 크기·색을 토큰으로 통제하고 각 아이콘에 `accessibilityLabel`을 답니다.

## 3. 단계별 디자인 프롬프트 (클로드코드용)

순서가 중요합니다 — **skin(토큰) → component(상태·밀도·터치) → layout(위계) → web/a11y → verify**. 회색 블록을 그대로 붙여넣으세요. 각 단계 끝에서 `npm test`와 `tsc`가 통과해야 다음으로 갑니다.

### 1단계 — 토큰 확장 (skin: 한 번에 전체 톤 전환)

```
디자인 시스템 토큰을 theme.ts에 도입해서 앱 전체 톤을 한 번에 바꾸자. 이번 단계는 토큰만 추가/정리하고, 컴포넌트 구조는 바꾸지 않는다(리스킨).

[src/theme/theme.ts 수정]
1. Theme 인터페이스에 브랜드 보조 토큰 추가: brandStrong, brandTint, brandBorder
   - light:  primary #3B5BDB, brandStrong #2C46B8, brandTint #EEF1FE, brandBorder #D8DFFB
   - dark:   primary #7C93FF, brandStrong #9DB0FF, brandTint #20283F, brandBorder #2E3A5C
   - onPrimary는 light #ffffff, dark #0B1020 유지
   - 의미색(danger/success/priority*)은 그대로 둘 것 — 브랜드와 겹치면 안 됨
2. 토큰 스케일 객체를 theme.ts에 export로 추가(테마 무관 상수):
   - type:   { xs:12, sm:13, md:15, lg:17, xl:20, xxl:24, xxxl:30, hero:40 }
   - weight: { body:'600', label:'700', number:'800' }
   - space:  { xs:4, sm:8, md:12, lg:16, xl:20, xxl:24, xxxl:32 }
   - radius: { sm:8, md:12, lg:16, pill:999 }
   - shadow: { sm: <RN 그림자 객체, 아주 옅게>, md: <모달용> }  // iOS/Android/web 호환 필드 포함
3. 숫자 표기 헬퍼: 숫자 텍스트에 쓸 공통 스타일 { fontVariant: ['tabular-nums'] } 를 export

[원칙]
- 색 값은 라이트/다크 둘 다 정의. 대비는 본문 4.5:1, 큰 텍스트/경계 3:1 이상 유지
- 이번 단계에서 화면 레이아웃·구조 변경 금지(토큰 추가와 기존 리터럴의 토큰 치환까지만)

먼저 theme.ts 변경안(diff)을 보여주고, 내 확인 후 적용. 적용 뒤 tsc/jest 통과를 확인해줘.
```

### 2단계 — 컴포넌트: 아이콘·칩·체크박스·상태

```
1단계 토큰 위에서 컴포넌트의 상태·밀도·터치 영역·아이콘을 정리하자. 기능과 핸들러는 그대로 두고 표현만 바꾼다.

[아이콘 통일]
1. @expo/vector-icons 도입(Expo 내장). 이모지(📅 🔁 ⏰ ⚑ 📁 👤 🌗 ☑ ☐ ✕ ＋)를 Feather/Ionicons로 교체.
   - 각 아이콘에 size/color를 토큰으로 지정, accessibilityLabel 부여
   - 매핑 예: 마감=calendar, 반복=repeat, 알림=bell, 우선순위=flag, 목록=folder, 하위=check-square,
     프로필=user, 테마=sun/moon, 삭제=x, 추가=plus

[TaskItem.tsx — 메타 칩 재설계(핵심)]
2. metaRow의 칩을 "색만으로 구분"에서 "형태로 구분"으로 바꾼다:
   - 미설정 칩: 외곽선(brandBorder) + 회색 아이콘/텍스트, 투명 배경
   - 설정 칩: brandTint 배경 + brand 텍스트/아이콘 (채움) — 아이콘+값+굵기로도 구분(색 단독 금지)
   - 칩 최소 높이 36~44, 좌우 padding 충분히 줘서 터치 영역 확보(hitSlop 대신 실제 크기)
   - radius는 pill, 라벨은 weight 700
3. 체크박스: 유니코드 글리프 대신 그린 원형 체크(View 24px 원 + 테두리, 완료 시 brand 채움 + check 아이콘),
   탭 타깃은 44x44 확보(패딩 포함). accessibilityRole='checkbox' 유지.
4. 마감 칩(dueChip): 지남은 dangerBg + danger + 'overdue' 아이콘/라벨 유지(색+형태 동시) — 그대로 좋음, 토큰만 적용.

[인증 화면 authStyles.ts]
5. 버튼/입력/배너에 토큰(radius.md, type, weight) 적용. 비밀번호 강도 바는 brand 단계색이 아니라
   중립→경고→성공 의미색으로(강도는 의미 신호). 구글 버튼은 외곽선 유지.

[공통 상태]
6. 모든 인터랙티브 요소에 press 상태(opacity 또는 brandTint), disabled 상태를 토큰으로 일관화.

먼저 TaskItem 칩/체크박스 before-after를 설명하고 구현. 적용 뒤 tsc/jest 통과 확인.
```

### 3단계 — 레이아웃: 위계 재배치 (오늘을 주인공으로)

```
정보 위계를 다시 잡자. 사용자가 앱을 열면 "오늘 얼마나 했는지"가 가장 먼저·크게 보여야 한다.

[TaskListScreen.tsx]
1. 헤더 위계 조정: 화면 최상단 글랜스 영역을 만들어 "오늘 진행"을 히어로로 승격.
   - 큰 수치: 오늘 완료/전체를 fsXxxl(30)~fsHero(40), weight 800, tabular-nums 로 표시
     예) "3 / 5" 또는 "{todayPct}%" 를 가장 크게, 그 아래 작은 라벨 "오늘 완료"
   - "할일" 타이틀은 보조로 축소하거나 상단 �’작은 워드마크로 이동
2. 진행 통계 카드: shadowSm 로 살짝 띄우고, 진행바 트랙=surfaceAlt / 채움=brand, 퍼센트는 tabular-nums.
   주간 완료 숫자도 number weight + tabular-nums.
3. 오늘/전체 토글: 활성 탭은 brand 텍스트 + 흰(또는 bg) pill, 비활성은 textMuted — 형태(채움)로도 구분.
4. add 바: 입력 radius.md, 추가 버튼은 brand 채움 + plus 아이콘 + '추가'. 마감 토글 칩은 2단계 칩 규칙 적용.
5. 간격을 space 스케일로 통일(16 좌우, 12 세로 리듬). 한 줄짜리 빈 상태 문구 유지.

[브랜드 모먼트]
6. 상단에 'my-plan' 워드마크(작은 텍스트 로고)를 brandStrong 색으로 한 곳에만. 과하지 않게.

먼저 새 위계의 레이아웃 스케치(영역 순서)를 글로 보여주고, 합의 후 구현. tsc/jest 통과 확인.
```

### 4단계 — 웹/접근성 마감 (react-native-web)

```
웹(react-native-web)과 접근성 마감을 하자.

1. 포커스 링: 웹에서 모든 Pressable·입력에 :focus-visible 가시 링을 준다.
   - 방법: 웹 전용 글로벌 CSS(예: web/index 또는 App 웹 엔트리)에서
     [tabindex], button, input:focus-visible { outline: 3px solid brandTint; box-shadow: 0 0 0 2px brand; }
     또는 Pressable에 Platform.OS==='web'일 때 outline 스타일 부여. 키보드 Tab으로 전체 흐름 점검.
2. 숫자 흔들림 방지: 카운트/퍼센트/주간완료에 tabular-nums가 실제 적용됐는지 확인.
3. prefers-reduced-motion 존중: 애니메이션(있다면) 축소 모드에서 비활성.
4. 대비 재확인: 본문 4.5:1, 큰 텍스트·경계 3:1. 특히 brand 위 흰 텍스트는 14px 이상 굵게.
5. 탭 타깃: 모바일 44x44 이상 최종 확인(칩·체크박스·아이콘 버튼).
6. 색 단독 의미 금지 최종 점검: 모든 상태(활성/지남/우선순위)가 색+아이콘/라벨/형태로 이중 표시되는지.

변경 후 build:web 성공과 tsc/jest 통과를 확인하고, 점검 결과를 체크리스트로 보고해줘.
```

### 5단계 — 검수 (squint test + 회귀)

```
디자인 적용을 검수하자.
1. Squint test: 창을 50%로 줄이거나 흐릿하게 봤을 때 (a) 주요 액션(추가/추가버튼) (b) 오늘의 핵심 수치
   (c) 경고/지남 상태가 즉시 구분되는지 확인. 안 되면 크기·대비·형태를 더 키운다(장식 추가 금지).
2. 토큰 일관성: 코드에서 남아있는 하드코딩 폰트크기/색/라운드 리터럴을 grep해 토큰으로 치환했는지 대조.
```
