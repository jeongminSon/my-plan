# 개인정보 처리방침 — my-plan

최종 업데이트: 2026-06-27

my-plan(이하 "앱")은 사용자의 프라이버시를 최우선으로 설계되었습니다. 이 문서는 어떤 정보를 수집·이용·보관·파기하는지 설명합니다.

## 1. 기본 동작 — 로컬 전용
앱은 **기본적으로 사용자의 기기에서만 동작**합니다.
- 할일·목록·하위 할일·마감일·반복·우선순위·메모는 **기기 로컬 저장소**(iOS/Android: SQLite, 웹: 브라우저 localStorage)에만 저장됩니다.
- 알림(리마인더)은 **로컬 알림**입니다. 푸시 토큰을 발급하지 않으며, 어떤 데이터도 외부로 전송하지 않습니다.
- **분석/광고 SDK를 사용하지 않습니다.** 크래시·오류 로그는 기기 내부에만 기록되며 외부로 전송되지 않습니다.

→ 로그인하지 않으면 **어떤 개인정보도 기기를 떠나지 않습니다.**

## 2. 클라우드 동기화(선택, 로그인 시에만)
여러 기기 간 동기화를 원할 때에 한해 소셜 로그인(구글/애플)으로 동기화를 켤 수 있습니다.

| 항목 | 내용 |
|---|---|
| **수집 항목** | 이메일, 소셜 로그인 식별자(sub) — 동기화에 필요한 **최소한**. 비밀번호는 수집·저장하지 않음 |
| **동기화 데이터** | 할일/목록의 제목·메모·마감일·완료여부·반복·우선순위·하위 할일 |
| **이용 목적** | 같은 계정으로 로그인한 기기 간 데이터 동기화 |
| **전송 보안** | 전 구간 TLS(HTTPS) 암호화 |
| **저장 보안** | 서버 저장 시 암호화(at-rest), 행 단위 접근통제(RLS)로 본인 데이터만 접근 |
| **보관 기간** | 계정 유지 동안. 계정 삭제 시 즉시 파기(아래) |
| **위탁/처리자** | 클라우드 백엔드 제공자(Supabase) — 동기화 저장 목적에 한함 |

## 3. 계정 및 데이터 삭제
앱 내 **"계정 삭제"** 로 언제든 직접 삭제할 수 있습니다. 삭제 시:
1. 서버의 할일/목록 데이터 즉시 삭제
2. 인증 계정 삭제
3. 기기 로컬 데이터·토큰 즉시 삭제
4. 자동 백업본은 백업 보관주기(최대 N일) 경과 시 자동 소멸 — 해당 기간을 본 방침에 명시합니다

## 4. 사용자 권리
- 데이터 열람·수정·삭제: 앱 내에서 직접 수행
- 동기화 중단: 로그아웃 시 즉시 중단(로컬 데이터는 유지)
